# PR 説明文ジェネレーター

git の差分情報から GitHub PR のタイトルと本文を生成するコマンド。

## 使い方

```
/pr-description [ベースブランチ名]
```

- 引数省略時は `main` をベースブランチとして使用する
- 例: `/pr-description` → main との差分でPRを生成
- 例: `/pr-description develop` → develop との差分でPRを生成

## 実行手順

### STEP 1: 差分情報を収集する

以下のコマンドを順に実行して差分情報を収集する。
ベースブランチは引数 `$ARGUMENTS` で指定されたもの、省略時は `main` を使う。

```bash
# 現在のブランチ名を取得
git branch --show-current

# コミット一覧（ベースブランチとの差分）
git log $BASE_BRANCH..HEAD --oneline

# 変更ファイル一覧
git diff --stat $BASE_BRANCH

# 変更内容の詳細（差分本文）
git diff $BASE_BRANCH
```

### STEP 2: 差分を分析する

収集した情報をもとに、以下の観点で変更内容を把握する。

- **変更の種別**: 機能追加 / バグ修正 / ドキュメント / リファクタリング / テスト / 設定変更
- **影響範囲**: どのレイヤー・集約に関係するか（domain / application / infrastructure / presentation）
- **変更の意図**: なぜこの変更が必要か（CLAUDE.md・コミットメッセージから推測）
- **破壊的変更の有無**: インターフェース変更・DB スキーマ変更など

### STEP 3: PR タイトルを生成する

以下のフォーマットで1行のタイトルを生成する。

```
<type>: <変更の要約>（50文字以内）
```

**type の選択基準（このプロジェクトの規則）:**

| type | 使う場面 |
|------|---------|
| `feat` | 新機能・新ファイルの追加 |
| `fix` | バグ修正・動作の修正 |
| `docs` | ドキュメントのみの変更（CLAUDE.md・Zenn記事・README など）|
| `chore` | 設定・ビルドの変更（コード変更なし）|
| `test` | テストのみの変更 |
| `ci` | CI/CD の変更 |

複数の type にまたがる場合は最も主要なものを選ぶ。

### STEP 4: PR 本文を生成する

以下のテンプレートを埋めて PR 本文を生成する。
不要なセクションは省略してよい（例: スクリーンショットが不要なら削除）。

```markdown
## 概要

<!-- この PR で何をしたか、なぜしたかを 3 〜 5 行で説明 -->

## 変更内容

<!-- 変更したファイル・クラス・関数を箇条書きで列挙 -->
<!-- ファイル名は backtick で囲む -->

### 実装の設計ポイント

<!-- DDD / クリーンアーキテクチャの観点で重要な設計判断があれば記載 -->
<!-- 例: DIパターン、集約の整合性保証、Result<T>の使い方 など -->

## テスト方法

<!-- DevContainer が必要な場合はその旨を明記 -->

```bash
# コンパイル確認
scripts/check-compile.sh

# スモークテスト
# （DevContainer 内で実行）
scripts/test-all.sh
```

## 関連 Issue

<!-- Closes #XX の形式で記載。Issue 番号が不明な場合は省略 -->

## チェックリスト

- [ ] コンパイルエラーなし（`check-compile.sh` PASSED）
- [ ] スモークテスト全件 PASSED（`test-all.sh`）
- [ ] `CLAUDE.md` の実装状況を更新済み（⬜ → ✅）
```

### STEP 5: ファイルに保存する

生成したタイトルと本文を `PR_DESCRIPTION.md`（プロジェクトルート）に書き出す。
ファイルフォーマットは以下の通り。

```markdown
# PR タイトル

（生成したタイトルをここに）

---

（生成した本文をここに）
```

### STEP 6: 結果を出力する

以下の形式で PR タイトルと本文を出力する。
コードブロックで囲み、そのままコピーして GitHub に貼り付けられる形にする。

---

**PR タイトル:**
```
（生成したタイトルをここに）
```

**PR 本文:**
```markdown
（生成した本文をここに）
```

---

出力後に「`PR_DESCRIPTION.md` に保存しました。コピーして GitHub の PR 作成画面に貼り付けてください」と一言添える。
修正が必要な場合はそのまま会話で調整できることを伝える。
